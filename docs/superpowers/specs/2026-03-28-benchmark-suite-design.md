# Benchmark Suite Expansion

Expand FerrFlow's benchmark suite from a basic 3-fixture bash loop to a comprehensive, hyperfine-based suite with stress testing, regression detection, and competitor comparisons.

## Goals

1. **Regression detection in CI** -- every push to main is compared against a baseline, CI fails if performance degrades beyond thresholds.
2. **Marketing / README** -- produce solid numbers showing FerrFlow's speed advantage over JS-based competitors at realistic and extreme scales.
3. **Stress testing** -- push FerrFlow to its limits (1000 packages, 50k commits) to find where it breaks.

## Fixtures

Five synthetic git repositories, generated by `benchmarks/fixtures/generate.sh`:

| Fixture | Packages | Commits | Purpose |
|---------|----------|---------|---------|
| `single` | 1 | 100 | Baseline, classic single-package repo |
| `mono-small` | 10 | 100 | Small monorepo |
| `mono-medium` | 50 | 500 | Realistic monorepo (renamed from current `mono-large`) |
| `mono-large` | 200 | 10,000 | Enterprise-scale monorepo |
| `mono-stress` | 1,000 | 50,000 | Stress test, find the breaking point |

Commits in large/stress fixtures have dates spread over 1 year with randomized scopes targeting random packages, simulating real history.

## Commands Benchmarked

### FerrFlow (all fixtures)

| Command | What it measures |
|---------|-----------------|
| `ferrflow check` | Full analysis: commit parsing, bump determination, changelog preview |
| `ferrflow release --dry-run` | Same path as check but via the release codepath |
| `ferrflow version` | Version file reading |
| `ferrflow tag` | Last tag lookup |

### Competitors (single, mono-small, mono-medium only)

| Tool | Command |
|------|---------|
| semantic-release | `npx --yes semantic-release --dry-run --no-ci` |
| changesets | `npx --yes @changesets/cli status` |
| release-please | `npx --yes release-please release-pr --dry-run` |

Competitors are excluded from mono-large and mono-stress -- they would take minutes, and the performance gap is already proven on smaller fixtures.

## Measurement Tool

**Hyperfine** replaces the manual bash timing loops. Each benchmark runs as:

```bash
hyperfine \
  --warmup 3 \
  --runs 10 \
  --export-json results/raw/<fixture>-<command>.json \
  'ferrflow check'
```

Hyperfine provides: min, max, mean, median, stddev per benchmark.

**Memory (RSS):** Measured separately with `/usr/bin/time -v`, one run per fixture/command combination. Hyperfine does not measure memory.

**Install size:** Measured for all tools:

| Tool | Method |
|------|--------|
| ferrflow (binary) | `du -m $(which ferrflow)` |
| ferrflow (npm) | `npm install ferrflow` then `du -sm node_modules` |
| semantic-release | `npm install semantic-release` then `du -sm node_modules` |
| changesets | `npm install @changesets/cli` then `du -sm node_modules` |
| release-please | `npm install release-please` then `du -sm node_modules` |

## Regression Detection

Two mechanisms:

### Relative threshold (single, mono-small, mono-medium, mono-large)

Compare the **median** of each benchmark against the corresponding median in `results/baseline.json`. Fail if `(new_median - baseline_median) / baseline_median > 0.25`. The 25% margin accounts for GitHub runner variance.

Binary size regression also checked: fail if the binary grows more than **20%** versus baseline.

### Absolute threshold (stress fixtures)

Fixed ceilings for extreme fixtures where relative comparison is unreliable:

| Fixture | Command | Max allowed |
|---------|---------|-------------|
| `mono-large` | `check` | 2s |
| `mono-large` | `version` / `tag` | 500ms |
| `mono-stress` | `check` | 10s |
| `mono-stress` | `version` / `tag` | 2s |

These thresholds are generous initially and should be tightened as historical data accumulates.

## File Structure

```
benchmarks/
├── fixtures/
│   ├── generate.sh          # extended with mono-large, mono-stress
│   ├── single/
│   ├── mono-small/
│   ├── mono-medium/         # renamed from mono-large
│   ├── mono-large/          # new: 200 packages, 10k commits
│   └── mono-stress/         # new: 1000 packages, 50k commits
├── results/
│   ├── baseline.json        # reference for regression, committed
│   ├── latest.json          # last run, committed on main
│   └── raw/                 # hyperfine JSON per benchmark, gitignored
├── compare.sh               # new: compares two result files, outputs diff
└── run.sh                   # rewritten to use hyperfine
```

### Output

- **stdout:** Markdown table with median, stddev, and size columns (same format as today, extended)
- **`results/raw/`:** One hyperfine JSON file per fixture/command combination
- **`results/latest.json`:** Aggregated results with all stats
- **exit code:** 0 if all checks pass, 1 if any regression detected

### `compare.sh`

Standalone script that takes two JSON files and produces a readable diff:

```bash
bash benchmarks/compare.sh results/baseline.json results/latest.json
```

Usable both locally and in CI.

## CI Integration

The `benchmark` job in `.github/workflows/ci.yml` changes:

```yaml
benchmark:
  name: Benchmark
  needs: test
  runs-on: ubuntu-latest
  if: github.event_name == 'push' && github.ref == 'refs/heads/main'
  steps:
    - uses: actions/checkout@v6
      with:
        fetch-depth: 0
    - uses: dtolnay/rust-toolchain@nightly
    - uses: Swatinem/rust-cache@v2
    - name: Install hyperfine
      run: cargo install hyperfine
    - name: Build ferrflow
      run: cargo build --release
    - name: Add ferrflow to PATH
      run: echo "$PWD/target/release" >> $GITHUB_PATH
    - uses: actions/setup-node@v4
      with:
        node-version: '22'
    - name: Generate fixtures
      run: bash benchmarks/fixtures/generate.sh benchmarks/fixtures
    - name: Run benchmarks
      id: bench
      run: |
        RESULTS=$(bash benchmarks/run.sh)
        echo "results<<BENCH_EOF" >> $GITHUB_OUTPUT
        echo "$RESULTS" >> $GITHUB_OUTPUT
        echo "BENCH_EOF" >> $GITHUB_OUTPUT
    - name: Check for regressions
      run: bash benchmarks/compare.sh benchmarks/results/baseline.json benchmarks/results/latest.json
    - name: Update baseline
      run: |
        cp benchmarks/results/latest.json benchmarks/results/baseline.json
        git add benchmarks/results/baseline.json
        git diff --cached --quiet || git commit -m "chore(bench): update baseline"
        git push
```

Key changes from current CI:
- `hyperfine` installed via cargo (fast with cache)
- Regression check step that can fail the job
- Baseline auto-updated after a passing run
- Fixture generation takes longer (mono-stress ~2-3 min) but only runs on push to main, never on PRs
